Framework Overview

The PHI-Nomos Framework applies Mandal, Gandhi & Siy's Modular Norm Model (Requirements Engineering, 2020) to HIPAA PHI gap analysis. Every HIPAA obligation is modelled as a super-situation containing one primary norm (duty or right), activation situations (conditions that make the norm active), and satisfaction situations (conditions that discharge the obligation). Compliance is computed by evaluating situations as TRUE/FALSE and propagating results upward through the norm hierarchy.

Five-Phase PHI-Nomos Process

Phase 1 — Regulatory Decomposition

Decompose HIPAA regulatory text (45 CFR Part 164, Subparts C/D/E) into atomic statements. Each becomes one super-situation. Use frame semantics templates to extract: holder, beneficiary, obligation, activation conditions, satisfaction conditions.

Phase 2 — Situation Inventory

For each super-situation, enumerate observable situational propositions (S-nnn). Each situation must be verifiable from evidence: policy docs, system logs, BAA records, training records, insurance certificates. Timestamp all situations.

Phase 3 — Scenario Definition

Evaluate each situation as TRUE/FALSE for the target date range using collected evidence. For retroactive analysis, use historical evidence from the gap period — not current state.

Phase 4 — Compliance Computation

Evaluate norm hierarchy: check activation → if active, check satisfaction → if active but unsatisfied, mark VIOLATED → propagate upward. Severity = Penalty Tier × Cascade Count × Gap Days.

Phase 5 — Gap Register & Remediation

Output structured gap register: norm ID, HIPAA citation, verdict, cascades, severity score, remediation action with priority. Suitable for OCR submission and remediation project planning.
